SSE — Solar Studio Engine

Python ECS game engine with hierarchical scene graph, rigid body physics, AABB collision, and 42 tools across 6 categories. Real-time 3D web viewer via Three.js + WebSocket.

game_engine/demo/sample_world.py
Python 
from game_engine.editor.app import EngineApp

# Initialize the engine
app = EngineApp()

# Create entities in the scene graph
app.execute_tool("create_entity", name="Player", parent_path="/Root")
app.execute_tool("add_component", path="/Root/Player",
    component_type="Mesh", properties={"mesh_type": "sphere", "color": [59, 130, 246]})
app.execute_tool("add_component", path="/Root/Player",
    component_type="RigidBody", properties={"mass": 1.0})
app.execute_tool("add_component", path="/Root/Player",
    component_type="Collider", properties={"shape": "sphere"})

# Add lighting
app.execute_tool("create_entity", name="Sun")
app.execute_tool("add_component", path="/Root/Sun",
    component_type="Light", properties={"light_type": "directional"})

# Run physics simulation
app.step(1/60)  # one frame at 60 FPS
state = app.get_world_state()  # → entities, frame, time

Entity-Component-System

Hierarchical scene graph with 11 component types: Transform, Mesh, RigidBody, Collider, Light, Camera, and more.

Physics Engine

Rigid body dynamics with gravity, AABB collision detection, velocity, forces, impulses, and raycasting.

42 Tools API

Entity, component, physics, query, scene, and scripting tools — the same API that AI agents use to control the world.

3D Web Viewer

Three.js-powered real-time viewport with shadows, lighting, particles — streamed via WebSocket at 30 FPS.

Product

SAS — Solar Agentic System

Agentic framework where everything is a Tool. LLM-orchestrated tool execution via OpenAI API, DAG-based parallel/serial workflows, YAML configuration, and three-layer context management.

agentic_system/configs/chatbot.yaml
YAML 
# Everything is a Tool — even the agent itself
tools:
  main:
    agent_v3:
      metadata:
        name: "main"
        description: "A helpful chatbot"
        input_schema:
          properties:
            task:
              string_schema:
                description: "User's question"
          required_fields: ["task"]
        output_schema:
          properties:
            response:
              string_schema:
                description: "Agent's response"
      instructions: |
        You are a helpful assistant.
      tools: [search, calculator]
      llm_config:
        model_name: "gpt-4o-mini"

Tool-Based Architecture

Everything is a Tool — agents, workflows, functions. The LLM decides which tools to call, yielding execution plans.

DAG Workflows

Declarative DAG execution with template expressions. Steps run in parallel when dependencies allow.

YAML Configuration

Define agents, tools, skills, and workflows in YAML. Support for !file includes and multi-agent composition.

Three-Layer Context

Local, parent, and global context layers with state persistence across multi-turn conversations.
